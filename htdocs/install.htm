Minimum System Requirements

  • The following Windows platforms are supported:
    • Microsoft® Windows 2000 Professional or Windows 2000 Server (SP4 or higher and security updates are recommended)
    • Microsoft Windows XP Professional (SP2)
    • Microsoft Windows VistaTM
    • Microsoft Windows Server 2003 (SP1)
    • The latest service packs and security updates are recommended for all platforms
    • Microsoft Internet Explorer 6.0 SP1 or later

      NOTE: Installing the English version of the product updates Internet Explorer with the English version of Internet Explorer 6.0 SP1. If you are running a localized operating system, run Windows Update to ensure that you get the proper localized version of Internet Explorer.

  • The product requires a number of prerequisites including the Microsoft .NET Framework 2.0 SDK, Microsoft .NET Framework 2.0 Redistributable Package, Microsoft Visual J#Version 2.0 Redistributable Package, Microsoft Data Access Components (MDAC) 2.8, and Microsoft Core XML Services (MSXML) 6.0. The prerequisites are automatically installed on your system during the install process if they are not already present. Although .NET 2.0 files are installed and required for the IDE to run, the product does not support development for .NET.
  • Free space requirement: Approximately 1.5 GB of disk space is required for full product installation, and the installer requires 2 GB additional temporary space.
  • Intel Pentium III/M 1.4GHz or Pentium IV 1.4 GHz minimum (Pentium III/M 1.4 GHz+ or Pentium IV 2GHz+ recommended).
  • 512 MB RAM (1 GB recommended)
  • DVD drive
  • XGA (1024x768) or higher resolution monitor
  • Mouse or other pointing device
  • The Together modeling functionality in the product requires the Java Runtime Environment. You can download the JRE from http://www.java.com.

General Installation Notes

Contents

The RAD Studio installation DVD contains C++Builder 2007 as well as Update 1 of Delphi 2007 for Win32. You can use the installation DVD to:

  • Install either product (Delphi 2007 for Win32, Update 1; or C++Builder 2007)
  • Install both products (Delphi 2007 for Win32, Update 1; and C++Builder 2007)
  • Upgrade an installation of Delphi 2007 for Win32 to Update 1.
Note that you can use the installation DVD for Delphi 2007 for Win32 only to install or upgrade Delphi 2007, but not to install C++Builder 2007.

On Windows VistaTM, must install as Administrator

For Windows Vista, you must run as Administrator or be on the Administrator list to install the product.

New installer, no bootstrap utility

Previous versions of Borland Developer Studio and Delphi included a bootstrap utility that checked for the presence of prerequisite software and installed it as necessary before the main product install started. Installation is now done by InstallAware, and there is no separate bootstrap utility; all is done within the new installer.

Multiple users on one machine

The product can be installed by any licensed user with administrator privileges, and is immediately available to all licensed users of the same machine. It is also available to any licensed users you might add later.

Installer might require a reboot

You are prompted to reboot your system at the end of the installation if the installer updated to Internet Explorer 6.0 SP1, or if the Microsoft .NET Framework v2.0 or MDAC 2.8 were installed from the installer setup.

Installer requires additional temporary space

If the drive containing the folder specified in your TEMP environment variable is low on space, the Installer might halt with an error message noting the shortage. If this occurs, you'll need to assign your TEMP variable to a folder on a drive with more disk space, then rerun the installation. See Minimum System Requirements for exact specification.

Assemblies installed in the Global Assembly Cache (GAC)

Duplicate copies of assemblies installed in the GAC are provided in the <CommonFilesFolder>\CodeGear Shared\RAD Studio\Shared Assemblies\5.0 directory. If C:\ is your Windows drive, the default <CommonFilesFolder> is C:\Program Files\Common Files\.

Verifying Installation of Update 1 for Delphi 2007 for Win32

To verify that your update has been installed, check the version number in the About box. The file timestamp is 11:00.

The only way to remove the update is to uninstall the product.

This update can be applied to all language editions.

Additional information

See the README.HTM file for additional information about the product.

Installing Either Product Using the Installation DVD (Delphi 2007 for Win32 (Update 1) or C++Builder 2007)

To install either product:
  1. Insert the installation DVD; this should auto-run the installer. If this does not happen, run Setup.exe from the install directory.
  2. When prompted, enter the serial number (license key) for the product.
  3. If you are prompted to select from Setup Personalities, select the product you are installing.

Installing Both Delphi 2007 for Win32 (Update 1) and C++Builder 2007 Using the Installation DVD

Use the appropriate procedure described below.

To install both products on a system that does not contain either product:

  1. Insert the installation DVD; this should auto-run the installer. If this does not happen, run Setup.exe from the install directory.
  2. When prompted, enter the serial number (license key) for the product you want to install.
  3. After the install completes, initiate a secondary installation to install the second product:
    • Select Start menu | Programs | CodeGear RAD Studio | Modify, Repair, Uninstall.
    • In the maintenance dialog, choose Upgrade.
    • Enter the serial number (license key) for the second product and complete the install.

To install both products on a system that already contains Delphi 2007 for Win32:

  1. Insert the installation DVD; this should auto-run the installer. If this does not happen, run Setup.exe from the install directory.
  2. When prompted, provide the appropriate serial number (license key) for C++Builder.
  3. When prompted for personalities to install, select both C++Builder and Delphi 2007 for Win32.

Upgrading Delphi 2007 for Win32 Using the Installation DVD

If you have the original release of Delphi 2007 for Win32 on your system, you can upgrade to Update 1 using the installation DVD. (See later section about electronic delivery for alternate media.) Setup.exe detects the earlier version of Delphi 2007 on your system and uninstalls that previous version and then install Delphi 2007 for Win32 Update 1.

To upgrade Delphi 2007 for Win32 to Update 1 on a system that already contains Delphi 2007 for Win32:

  • Using the installation DVD, run install\setup.exe with the /upgrade switch. The /upgrade switch means that the installer displays fewer dialogs and installs the same edition of Delphi 2007 for Win32 that was previously on your system.

Installing via Electronic Software Delivery

If the product is purchased via Electronic Software Delivery, you receive a setup.exe file.

To install using Electronic Software Delivery:

  1. Double-click the setup.exe file to start the installation.
  2. When prompted, enter your serial number (license key).
  3. All the required ZIP files are downloaded from the Internet and uncompressed.

  4. You are asked to supply an archive location for the install ZIP files.

    Note: Backing up the downloaded ZIP files is strongly encouraged so that in the future you can do a reinstall without downloading all the ZIP files again.

  5. The install proceeds automatically.

To install the second product:

If you have installed C++Builder 2007 or Delphi 2007 for Win32 Update 1, and you want to install the second product but you do not have an installation DVD, you can run a secondary or Maintenance install, which is equivalent to running setup.exe from the DVD. Use the following steps:

  1. Select Start> Programs | CodeGear RAD Studio | Modify, Repair, Uninstall.
  2. In the Maintenance dialog, choose to Upgrade your current installation.
  3. Enter the new serial number (license key) for the second product.

Upgrading Delphi 2007 for Win32 via Electronic Software Delivery

If you have Delphi 2007 for Win32 and you want to upgrade to Delphi 2007 for Win32 Update 1:

  1. Select Start | Programs | CodeGear RAD Studio | Check for updates.
  2. In the list of available updates, select Update 1.
  3. Your current Delphi 2007 for Win32 installation is uninstalled, and then the required files are downloaded and installed automatically.

Note: Due to the MSI registration and unregistration that take place, this upgrade process might take an hour or two hours. If you have problems with the download, you can restart the install by running setup.exe /upgrade, and the install attempts to start from where it ended.

Installing Using Purchased Network Licenses

If you purchased Networked Licenses (managed by the Borland License Server or FLEXlm) for Delphi 2007 for Win32 or C++Builder 2007, you must follow these instructions to set up the install image or your users will be prompted for a serial number and be unable to install.
  1. Copy all the files from your media kit DVD to a writeable directory on your network.
  2. Copy the networked license .slip file you received in your order confirmation email to the \Install directory where setup.exe is located.
  3. When your users run setup.exe, the product detects the .slip file and installs the software. During installation, the .slip file is moved to the license folder in the installation directory.

Known Installation Issues

  • If you modify the CodeGear product installation by clicking the Change button in Add/Remove Programs and choosing the Modify option to remove features, setup makes the appropriate changes in the registry only in HKEY_LOCAL_MACHINE. Follow any one of the steps below to make these changes in HKEY_CURRENT_USER.
    • After doing a maintenance install from Add/Remove Programs, restart the IDE and click on No for all the Package not found errors and Assembly not found errors. This automatically removes the correct assembly or package entries from the registry and should prevent these errors when starting CodeGear Delphi 2007. This is the preferred option, since the options outlined below involve manually editing the registry.
    • Manually edit the HKEY_CURRENT_USER\Software\Borland\BDS\5.0\Known IDE Packages and Known Packages registry keys to delete the assemblies or packages that were removed by the setup in HKEY_LOCAL_MACHINE\Software\Borland\BDS\5.0.
    • Delete the HKEY_CURRENT_USER\Software\Borland\BDS\5.0 registry key and restart the IDE to get the updated registry settings. Be aware that when you delete this key, you lose your customized settings that were stored under this registry key.

      Note: Read the WARNING at the end of this file before editing your system registry.

  • Uninstalling Delphi 2007 when an earlier version of BDS or C++Builder is installed should not affect pre-existing installations.

  • You cannot have Delphi 2007 installed and install the C++ Trial version or vice versa. The personality associated with the trial will not have the files installed.
  • If you have a Delphi Trial installed and install C++Builder Enterprise or vice versa, your Delphi Trial will be uninstalled.
  • You cannot install the previous Delphi 2007 version over an installation of C++Builder 2007 or Delphi 2007 for Win32 (Update 1). If you want to reinstall Delphi on a system with C++Builder 2007, you must use the Delphi install from this DVD (Update 1).
  • The Installer does not notify you if you have less than the required free space (2 GB on the C: drive). If you try to install with too little space, the install hangs.

Uninstalling This Product

  • To uninstall the product, the Administrator who initially installed the product must open the Control Panel, select Add or Remove Programs, select RAD Studio and click the Remove button. After uninstalling, in order to remove user-specific data from the Windows registry, all non-Administrator users of the product must log in and delete the following registry key:

    HKEY_CURRENT_USER\Software\Borland\BDS\5.0

    WARNING: Registry editing is performed at your own risk. This document and other documentation that accompanies this product describe how to edit your Windows registry. In most cases, CodeGear recommends that you use the Microsoft Regedit tool to perform editing. Whatever tool you use, however, you should, before making any changes, make backups of your registry and thoroughly familiarize yourself with the editing tool and the registry editing process. The registry is a critical component of your operating system, and any mistakes you make while editing it can cause your machine to stop functioning.

  • After uninstalling, user-specific data such as default project settings might remain in configuration files found in this directory:

    Documents and Settings\<username>\Application Data\Borland\BDS\5.0\

    This folder should be manually removed.
